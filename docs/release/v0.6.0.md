# iqdb-flat v0.6.0 — Examples + entering the validation band

**Into the home stretch.** The index has been feature-complete since v0.4.0 and
API-frozen since v0.5.0. v0.6.0 opens the pre-1.0 validation band (alpha) and
ships the runnable example suite. There is no API change — this release is
purely additive.

## What's in 0.6.0

Four worked examples, each runnable with `cargo run --example <name>` and each
asserting its own output so they stay correct:

- **`quick_start`** — the shortest end-to-end path: build a `FlatIndex`, insert,
  search, read `stats`.
- **`metric_tour`** — one query under all five metrics (Cosine, DotProduct,
  Euclidean, Manhattan, Hamming), demonstrating the single *smaller-is-nearer*
  ordering contract and the `DotProduct` negation.
- **`filtered_search`** — metadata pre-filtering with a compound
  `lang == "rust" AND year > 2020` filter, including the closed-world rule
  (a row with no metadata never matches a positive predicate).
- **`recall_oracle`** — flat's reason for existing: it measures **recall@k** of a
  stand-in pruned index against flat's exact top-`k`, the same way HNSW and IVF
  will be measured against it.

## Breaking changes

**None.** The public API is unchanged from v0.4.0/v0.5.0 and remains frozen for
the 1.x series.

## Verification

Clean across the CI matrix (Linux, macOS, Windows) on stable and the 1.87 MSRV,
default features and `--all-features`; every example builds, runs, and asserts:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
cargo run --example quick_start
cargo run --example metric_tour
cargo run --example filtered_search
cargo run --example recall_oracle
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
```

## Where this sits on the road to 1.0

flat's own work is essentially done: the feature set is complete, the API is
frozen, correctness is pinned by property tests and an exact large-scan oracle,
hot paths are benchmarked, and there is zero `unsafe`. `loom` is **not
applicable** — flat has no lock-free or shared-mutable path (it is
single-writer-internal; the engine guards it with an external `RwLock`), so there
is nothing for a concurrency model checker to explore.

What remains before `1.0.0` is **external validation**: exercising flat as the
recall oracle against the real approximate indexes (`iqdb-hnsw`, `iqdb-ivf`) as
those crates mature, plus final cross-crate benchmarks. That work lives in the
consumer crates and `iqdb-eval`, not here. 1.0.0 is the stability commitment once
that integration has run.
