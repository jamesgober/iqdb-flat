# iqdb-flat v1.0.0 — Stable

**The exact index is committed.** Six 0.x releases built it, froze its surface,
and proved it correct; v1.0.0 puts it under the SemVer 1.x guarantee — no
breaking changes until 2.0. There is no new public API here. This is the
stability promise: what you build against today keeps compiling.

## What is iqdb-flat?

The brute-force **exact** nearest-neighbour layer of the iQDB vector database. It
scans every stored vector on every query, computes the true distance, and
returns the top-`k` — always correct, by construction. That makes it two things
at once: a real index for small corpora (under ~10k vectors, where graph or
partition overhead isn't justified) and the **recall ground truth** every
approximate index (HNSW, IVF) is measured against. It is the first and simplest
implementer of the `iqdb-index` `Index` trait, so it also validates that trait's
design.

## The 1.0 surface

```rust
use std::sync::Arc;

use iqdb_flat::{FlatConfig, FlatIndex};
use iqdb_index::{Index, IndexCore};
use iqdb_types::{DistanceMetric, SearchParams, VectorId};

# fn main() -> iqdb_types::Result<()> {
let mut idx = FlatIndex::new(2, DistanceMetric::Euclidean, FlatConfig)?;
idx.insert(VectorId::from(1u64), Arc::<[f32]>::from(&[0.0, 0.0][..]), None)?;
idx.insert(VectorId::from(2u64), Arc::<[f32]>::from(&[3.0, 4.0][..]), None)?;
idx.insert(VectorId::from(3u64), Arc::<[f32]>::from(&[1.0, 0.0][..]), None)?;

let hits = idx.search(&[0.0, 0.0], &SearchParams::new(2, DistanceMetric::Euclidean))?;
assert_eq!(hits[0].id, VectorId::U64(1));
assert_eq!(hits[1].id, VectorId::U64(3));
# Ok(())
# }
```

The committed surface (frozen, recorded in `dev/ROADMAP.md`):

- **`FlatIndex`** — `new` (via `Index`), `new_unconfigured`, `dim`, `metric`,
  `len`, `is_empty`, plus the full `IndexCore` implementation (`insert`,
  `insert_batch`, `delete`, `search`, `search_batch`, `flush`, `stats`).
- **`FlatConfig`** — the unit configuration type.
- **`VERSION`** and the **`Hit`** re-export.
- **Feature `parallel`** — internal-only; does not change the API surface.

## Why this is 1.0

Every Definition-of-Done criterion (`dev/DIRECTIVES.md` §7) is met:

- **Clean everywhere** — `fmt`, `clippy -D warnings` (default and all features),
  `test --all-features`, and `cargo doc -D warnings` pass on the CI matrix
  (Linux, macOS, Windows), on stable and the 1.87 MSRV.
- **Secure** — `cargo audit` + `cargo deny` clean; **zero `unsafe`**
  (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` / `todo!` / `dbg!` in
  shipping code; every allocation bounded; no method panics on hostile input
  (non-finite floats, over-deep filters, and `usize::MAX` `k` all return a clean
  `Result`, pinned by `tests/error_hardening.rs` including a no-panic property
  test).
- **Correct** — flat is the oracle, so correctness is cross-checked against an
  independent hand-coded reference for all five metrics, property-tested, and
  pinned **bit-for-bit at N = 20,000** by `tests/large_scan.rs`. The stable
  insertion-order tiebreaker survives `swap_remove` deletes and
  delete-then-reinsert churn.
- **Proven sufficient** — the `recall_oracle` example builds the exact-vs-pruned
  recall measurement flat exists to provide, on the public API alone.
- **Documented** — every public item carries a runnable rustdoc example; the
  README leads with the Tier-1 surface; `docs/API.md` covers the whole API; eight
  runnable examples ship in `examples/`.
- **Benchmarked** — `criterion` benches cover the hot paths: unfiltered search,
  filtered search, and bulk insert.

`loom` is **not applicable**: `FlatIndex` has no shared-state or lock-free path.
It is single-writer-internal (the engine guards it with an external `RwLock`), so
there is nothing for a concurrency model checker to explore.

## Breaking changes

**None.** The public API is unchanged from v0.5.0/v0.6.0; this release is the
stability commitment.

## What's next (post-1.0, additive)

- Continued validation as the recall oracle against `iqdb-hnsw` / `iqdb-ivf` as
  those crates mature, via `iqdb-eval` — additive, and outside flat's frozen API.
- A contiguous row-major storage layout remains a tracked internal optimisation;
  being internal, it can land in a 1.x without breaking anything.
